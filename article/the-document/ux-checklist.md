# UX Checklist

## The UX Audit Framework <a href="#the-ux-audit-framework" id="the-ux-audit-framework"></a>

Below are the core principles I review for every screen or flow.

| User Understanding | User-Centered           | Does this screen clearly support the user’s goal? |
| ------------------ | ----------------------- | ------------------------------------------------- |
| Clarity            | Clear Next Step         | Can users immediately tell what to do next?       |
| Consistency        | UI Consistency          | Are components aligned with the design system?    |
| Feedback           | Interaction Feedback    | Does every action produce a response?             |
| System Status      | Visibility              | Do users know what the system is doing?           |
| Cognitive Load     | Simplicity              | Is the screen overloaded with information?        |
| Decision Making    | Hick’s Law              | Are there too many choices at once?               |
| Interaction        | Fitts’s Law             | Are tap targets large and easy to reach?          |
| Memory             | Recognition over Recall | Do users recognize instead of remember?           |
| Error Handling     | Prevention & Recovery   | Are errors prevented and easy to fix?             |
| Navigation         | Orientation             | Do users always know where they are?              |
| Efficiency         | Flow Optimization       | Are there unnecessary steps?                      |
| Visual Hierarchy   | Focus                   | Is the primary action visually dominant?          |
| Perception         | Aesthetic–Usability     | Does the UI feel trustworthy and polished?        |
