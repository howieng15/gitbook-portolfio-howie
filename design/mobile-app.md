## TM Wallet

**TM Wallet** is a secure digital wallet designed for managing crypto assets, enabling users to create and recover wallets, send and receive tokens via QR codes, and interact with blockchain transactions through a streamlined, user-friendly mobile experience.

<details>

<summary>Read more</summary>

### **Overview** <a href="#overview" id="overview"></a>

Design crypto wallet app.

### **Challenges** <a href="#challenges" id="challenges"></a>

* Developing a concept that stands out in a saturated market.
* Ensuring the design is unique yet familiar to users.
* Implementing a dark theme that is visually appealing and functional.

### **Solution** <a href="#solution" id="solution"></a>

To address these challenges, I analyzed existing crypto wallets to understand common design patterns and user expectations. I then introduced a fresh visual style, combining modern elements with a sleek, dark theme. The result was an interface that is both attractive and user-friendly, ensuring it stands out while maintaining usability.

### **Takeaways** <a href="#takeaways" id="takeaways"></a>

This project significantly enhanced my design skills, particularly in translating wireframes into a polished UI. It also gave me deeper insight into designing for financial applications and creating visually distinctive yet practical designs.

## AI Task Discovery System

A design test exploring how to help users discover AI-use cases. The solution is a single-screen interface that supports task identification through intuitive input and AI-assisted suggestions.

<details>

<summary>Read more</summary>

### Design Assignment – AI Task Discovery Interface

#### **Context**

A UX/UI expert receives quantitative research insights indicating that:

> The main barrier preventing users from adopting AI is their inability to identify and list tasks where AI can be applied.

***

#### **Goal**

Design **a single-screen interface** that enables users to:

* Discover
* Identify
* Complete a **list of tasks** where AI can assist

***

#### **Success Metrics**

Evaluation is based on:

* **Total number of tasks added to the list**

***

#### **Counter Metrics**

To ensure quality and usability:

* **% of important tasks / total tasks ≥ 80%**\
  → Important tasks should dominate the list
* **Time spent per action**\
  → Minimize user effort and friction
* **Click-through rate (CTR) on CTA**\
  → Ensure the call-to-action is effective

***

#### **Constraints & Requirements**

* The **entire task list must be viewable within a single screen**
* The **input method must be intuitive and easy to understand**

***

#### **Design Considerations**

Careful consideration is required for:

**1. Data Input Methods**

* Active input (manual task entry)
* Passive input (AI suggestions, prompts, recommendations)

**2. Data Presentation**

* Structuring and grouping tasks
* Highlighting **important vs. non-important tasks**
* Supporting quick scanning and comprehension

***

#### **Core Sections (Suggested Structure)**

The interface should include **three main components**:

1. **Task Input Section**
   * Active input (user-driven)
   * Passive input (AI-assisted suggestions)
2. **Task Visualization Section**
   * Clear, scannable task list
   * Prioritization or categorization
3. **Call-To-Action (CTA)**
   * Encourages progression or confirmation

***

#### **Expectation**

The solution should demonstrate:

* Strong thinking in **input mechanisms**
* Effective **information presentation and prioritization**
* Clear **design rationale and decision-making**
